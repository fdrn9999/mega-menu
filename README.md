# mega-menu

메가스터디 구내식당 메뉴 조회 디스코드 봇입니다.
네이버 블로그에서 이번 주 식단표 이미지를 가져와, 평일에는 오늘 요일의 점심 메뉴만 잘라서 보여줍니다.

## 명령어

| 명령어 | 설명 |
|---|---|
| `/메가스터디` | 오늘의 점심 메뉴 (주말에는 다음 주 전체 메뉴표) |
| `/디버그` | [서버 오너 전용] 이미지 URL과 월~금 크롭 결과 확인 |

## 파일 구성

```
app.py             # 봇 본체 (디스호스트 기본 시작 파일명)
requirements.txt   # 의존성 목록
.env               # 봇 토큰 (DISCORD_TOKEN=...) — 절대 커밋 금지
cache/             # 봇이 자동 생성하는 주차별 식단표 캐시
```

## 토큰 설정 (우선순위 순)

1. **시스템 환경변수**: `DISCORD_TOKEN` (호스팅 패널이 지원하는 경우)
2. **.env 파일(기본)**: `app.py` 와 같은 폴더의 `.env` 에 `DISCORD_TOKEN=토큰` 작성
3. **token.txt**: 토큰만 한 줄 입력 (폴백)

⚠️ 토큰을 코드에 직접 쓰거나 깃에 커밋하지 마세요. 노출됐다면 즉시
[Discord Developer Portal](https://discord.com/developers/applications) → Bot → **Reset Token** 으로 재발급.

## 디스호스트 배포

1. GIT_ADDRESS 에 이 저장소 주소를 넣으면 서버가 자동으로 클론/업데이트 (시작 파일 기본값 `app.py` 그대로 사용)
2. `.env` 는 깃에 없으므로 **패널의 파일 관리자에서 직접 생성**: 내용은 `DISCORD_TOKEN=토큰` 한 줄
3. 봇 시작 — 콘솔에 `✅ 로그인 성공` 이 뜨면 완료

`.env` 는 gitignore 파일이라 자동 업데이트(`git reset --hard`)에도 지워지지 않고 유지됩니다.

코랩과 달리 24시간 프로세스가 유지되므로 keep-alive 웹서버는 필요 없습니다.

## 저사양(128MB RAM / CPU 25% / 512MB 디스크) 최적화

- 크롤링·다운로드·크롭은 **주차당 1번**만 실행, 결과를 `cache/` 폴더에 저장
- 명령어 호출 시에는 미리 잘라둔 작은 PNG 파일만 읽어 전송 → CPU/RAM 거의 사용 안 함
- 이미지 후보 URL은 `Content-Length` 로 크기만 비교 후 **1개만** 다운로드 (상한 12MB)
- discord.py 메시지/멤버 캐시 비활성화로 상주 메모리 절약
- 새 주차 캐시 생성 시 이전 주차 파일 자동 삭제

## 로컬 테스트

```
pip install -r requirements.txt
python main.py
```
